# 30-Day Python Learning Journey (1 Hour/Day)

**TL;DR - Master Python fundamentals through structured daily practice, progressing from basics to building real projects. Start with existing workspace examples, then expand to new concepts with hands-on coding and verification.**

## Steps

1. **Day 1: Python Setup & Hello World** - Install Python, set up IDE (VS Code), run hello_world.py, understand print statements and basic syntax. *Verification: Write and run a simple "Hello, World!" program.*
2. **Day 2: Variables & Data Types** - Study strings, numbers, booleans in hello_world.py. Practice variable assignment and basic operations. *Verification: Create a program that declares variables of different types and prints them.*
3. **Day 3: String Manipulation** - Explore hello_ways.py for 15+ string techniques (f-strings, format(), concatenation). *Verification: Write a program using at least 3 different string formatting methods.*
4. **Day 4: User Input & Basic Operations** - Learn input() function, type conversion. Practice with calculator.py basics. *Verification: Build a simple calculator that takes two numbers and an operation.*
5. **Day 5: Conditional Statements** - Study if/elif/else in calculator.py. Practice decision-making logic. *Verification: Create a program that categorizes user age into child/teen/adult.*
6. **Day 6: Loops** - Examine loops in calculator.py. Learn for/while loops, break/continue. *Verification: Write a program that prints multiplication tables using loops.*
7. **Day 7: Functions** - Deep dive into function definitions and calls in calculator.py. *Verification: Create a function library with basic math operations.*
8. **Day 8: Lists & Tuples** - Learn list operations, indexing, slicing. Create new file: data_structures.py. *Verification: Build a program that manages a shopping list with add/remove operations.*
9. **Day 9: Dictionaries & Sets** - Study dict/set creation, methods. Add to data_structures.py. *Verification: Create a phone book application using dictionaries.*
10. **Day 10: List/Dict Comprehensions** - Learn concise list/dict creation. Practice with examples. *Verification: Rewrite previous programs using comprehensions.*
11. **Day 11: Exception Handling** - Study try/except in calculator.py. Learn custom exceptions. *Verification: Add error handling to your calculator from Day 4.*
12. **Day 12: File I/O Basics** - Learn reading/writing text files. Create file_operations.py. *Verification: Build a simple note-taking app that saves to file.*
13. **Day 13: Modules & Imports** - Study importing modules, creating custom modules. *Verification: Split your calculator into separate modules.*
14. **Day 14: Working with Dates & Times** - Use datetime module. Practice formatting. *Verification: Create a program that calculates age from birthdate.*
15. **Day 15: OOP Fundamentals** - Study classes in addition.py (AbstractAdder, NumberValidator). *Verification: Create a simple class for a Book with attributes and methods.*
16. **Day 16: Inheritance** - Learn inheritance patterns from addition.py (BasicAdder → AdvancedAdder). *Verification: Extend your Book class with subclasses (e.g., EBook).*
17. **Day 17: Encapsulation & Properties** - Study private attributes and properties in addition.py. *Verification: Add encapsulation to your Book classes.*
18. **Day 18: Polymorphism** - Understand method overriding from ScientificAdder in addition.py. *Verification: Implement polymorphism in your book hierarchy.*
19. **Day 19: Advanced OOP Concepts** - Study mixins, composition, operator overloading in addition.py. *Verification: Add operator overloading to your classes.*
20. **Day 20: Regular Expressions** - Learn re module for pattern matching. *Verification: Create an email validator using regex.*
21. **Day 21: JSON & Data Serialization** - Work with json module, API-like data handling. *Verification: Build a program that reads/writes JSON data.*
22. **Day 22: Project Planning** - Design a small project (e.g., task manager or weather app). *Verification: Write project requirements and basic structure.*
23. **Day 23: Building the Project Core** - Implement main classes/functions for your project. *Verification: Get basic functionality working.*
24. **Day 24: Adding Features** - Extend project with additional features (file storage, user interface). *Verification: Test all new features.*
25. **Day 25: Error Handling & Testing** - Add comprehensive error handling and basic tests. *Verification: Write unit tests for key functions.*
26. **Day 26: Code Refactoring** - Clean up code, improve structure, add documentation. *Verification: Ensure code follows PEP 8 standards.*
27. **Day 27: External Libraries** - Learn pip, install and use requests library. *Verification: Integrate an API call into your project.*
28. **Day 28: Final Project Touches** - Add logging, configuration, polish UI. *Verification: Project runs smoothly with good user experience.*
29. **Day 29: Code Review & Optimization** - Review all code, optimize performance. *Verification: Measure and improve execution time.*
30. **Day 30: Showcase & Next Steps** - Present your project, plan future learning. *Verification: Demo project and document what you learned.*

## Relevant files
- `hello_world.py` — Basic syntax foundation (Days 1-2)
- `hello_ways.py` — String manipulation techniques (Day 3)
- `calculator.py` — Functions, input, conditionals, loops (Days 4-7)
- `addition.py` — Advanced OOP concepts (Days 15-19)
- `data_structures.py` — New file for lists/dicts/sets (Days 8-9)
- `file_operations.py` — New file for I/O operations (Day 12)
- Project files — Custom files for Days 22-30

## Verification
1. **Daily Code Execution** - Run each day's code and ensure no syntax errors
2. **Output Validation** - Check that programs produce expected results with sample inputs
3. **Code Review** - At end of each week, review code for best practices
4. **Project Milestones** - Days 22-30: Test project functionality incrementally
5. **Final Assessment** - Day 30: Project demo and self-evaluation quiz

## Decisions
- **Beginner Focus** - Assumes no prior programming experience, building from workspace examples
- **Progressive Difficulty** - Weeks 1-2: Basics, Weeks 3-4: Intermediate, Week 5: Project
- **Practical Emphasis** - Each day includes hands-on coding, not just theory
- **Workspace Integration** - Leverages existing files for first week, then expands
- **Excluded Scope** - Advanced topics like web frameworks, databases, machine learning (save for future plans)

## Further Considerations
1. **Pace Adjustment** - If concepts take longer than 1 hour, extend to 1.5 hours rather than rushing
2. **Practice Resources** - Supplement with free online platforms like LeetCode, HackerRank for extra exercises
3. **Community Support** - Join Python communities (Reddit r/learnpython, Discord servers) for questions
4. **Tool Setup** - Ensure VS Code with Python extension, virtual environments for Day 27+</content>
